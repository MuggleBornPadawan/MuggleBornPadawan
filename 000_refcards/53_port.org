* type systems
** intro
- a type system is fundamentally about boundaries and predictability
  - abstraction level: replacing memory addresses with everyday metaphors (boxes, labels, contracts)
  - precision vs. intuition: balancing easy-to-grasp analogies with structural, mathematical realities
  - operational risk: shifting from "why does this matter?" to "how does this break at scale?"
** notes
- enforceable design contract (ieee standards)
  - space allocation and encoding
  - operational domain - set of valid operations (eg: arithmetic operations on numbers; concat for text)
