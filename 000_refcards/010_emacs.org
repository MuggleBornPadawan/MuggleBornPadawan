GNU emacs
** getting started
- C-h C-a open GNU Emacs manual 
- M-x toggle-frame-fullscreen
- C-x C-c quit emacs
- C-g quit command
- C-x C-+ increase font size
- C-x C—- decrease font size 
- C-h <command> help 
- C-h v variables
- C-h a Apropos 
- Esc-Esc-Esc - all purpose “get out” command
** initial setup
- edit .emacs.d/init.el
- (toggle-frame-fullscreen) | full screen 
- package archives: gnu, melpa
- packages: magit, doom-themes, others 
- (add-to-list 'default-frame-alist '(font . "Menlo-26"))
- (global-display-line-numbers-mode t) ;; set line numbers
- (column-number-mode t) ;; Enable column numbers
- (menu-bar-mode -1)       ;; Disable the menu bar
- (tool-bar-mode -1)       ;; Disable the tool bar
- (scroll-bar-mode -1)     ;; Disable the scroll bar
- (setq backup-directory-alist `(("." . "~/.emacs.d/backups"))) ;; Configure backups
- (global-set-key (kbd "C-c R") 'revert-all-buffers)  ;; Bind to Ctrl + c, Shift + r
- (list-buffers)
- (global-visual-line-mode 1) ;; soft wrap text globally
- (recentf-mode 1) ;; recent files history is saved 
- (savehist-mode 1) ;; recent commands history is saved | use Mn (next-history-element) Mp (previous-history-element) 
- (setq history-length 25) ;; saves n recent commands 
- (save-place-mode 1) ;; saves cursor location on files 
- (global-auto-revert-mode 1) ;; refreshes all buffers
- (setq global-auto-revert-non-file-buffers t) ;; refreshes non-file buffers (eg: folders)
- (split-window-vertically)
- (view-lossage) ;; C-h l

*** Document Exporting (PDF Export)
To test exporting this document to a PDF using =latexmk=, run the export menu in Emacs:
1. Press =C-c C-e= to open the Org Export Dispatcher.
2. Press =l= to select the LaTeX export sub-menu.
3. Press =p= to export to a PDF file (=latex_test.pdf=), or press =o= to export to PDF and open the file immediately.

** marks
*** The Mark Ring (Quickest)
Best for jumping back to a spot you just left.
- **Mark:** Press `C-SPC` (sets the mark at your current position).
- **Go back:** Press `C-u C-SPC` (moves point back to the mark). You can repeat `C-u C-SPC` to cycle backward through previous marks.
*** Registers (Multiple named spots)
Best for saving a specific spot while you work elsewhere.
- **Mark:** Press `C-x r SPC` followed by a single character (e.g., `a`).
- **Go back:** Press `C-x r j` followed by the same character (`a`).
*** Bookmarks (Persistent across sessions)
Best for long-term reference.
- **Mark:** Press `C-x r m` and give it a name.
- **Go back:** Press `C-x r b` and select your bookmark.
*** Removing Bookmarks (Persistent)
- **Quick way:** Type `M-x bookmark-delete` and press `RET`, then enter the name of the bookmark to delete.
- **Visual way:** 
  1. Open the bookmark list with `C-x r l`.
  2. Move the cursor to the bookmark you want to delete.
  3. Press `d` (flags it for deletion).
  4. Press `x` (executes the deletion).
*** "Removing" Registers (Single-character)
- In Emacs, registers are temporary and lightweight, so there is no built-in command to delete them. Instead, you simply **overwrite** them by saving a new location to the same character (e.g., `C-x r SPC a`). 
- If you want to clear a register completely, you can run:
  - `M-x set-register` -> enter the register key -> enter `nil`.

*** header arguments
#+NAME: top_10_org_babel_headers
|-----------------+-----------------------------------+---------------------------------------------------------------------------------------|
| Header Argument | Common Permutations / Values      | Operational Description                                                               |
|-----------------+-----------------------------------+---------------------------------------------------------------------------------------|
| :results        | value, output, table, silent, raw | Controls how code output is collected and rendered back into the Org buffer.          |
| :session        | none, *name* (e.g., *py*, *clj*)  | Runs code in a persistent, stateful REPL process instead of an ephemeral instance.    |
| :exports        | code, results, both, none         | Determines what elements are included when publishing/exporting the document.         |
| :tangle         | no, yes, filename.ext             | Extracts code into an external script file during a tangle operation (`C-c C-v t`).   |
| :var            | name=value, name=table_name       | Binds data variables, properties, or table outputs directly into the code scope.      |
| :dir            | /path/to/dir, /ssh:user@host:/dir | Sets the default working directory for execution; supports remote Tramp paths.        |
| :cache          | yes, no                           | Skips code re-execution unless the source block's hash or inputs have changed.        |
| :noweb          | no, yes, strip-export             | Enables literate programming expansions using the <<macro_or_block_name>> syntax.     |
| :colnames       | yes, no                           | Specifies whether the first row of an incoming input table should be column headers.  |
