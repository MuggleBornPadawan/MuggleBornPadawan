** software development lifecycle
*** software engineer
- definition - software engineering is the application of scientific principles to the design and creation of software. The field uses a systematic approach to collect and analyze business requirements in order to design, build, and test software applications to satisfy those business requirements
- computer aided software engineering (case, 1980s) - business analysis and modeling, development tools, such as debugging environments, verification and validation tools, configuration management, metrics and measurement, and project management
- responsiblities - include design, build and maintain software systems. writing and testing code and consulting with stakeholders such as clients, third-party software vendors, security specialists, and other team members.
- developer (broad - structure)  vs engineer (narrow - specific)

**** requirements
- steps - identifying stakeholders, establishing goals and objectives, eliciting requirements from the stakeholders, documenting the requirements, analyzing and confirming the requirements, and prioritizing

- classification - must have, desired, nice to have. collection mode: survey, questionairre, interviews
- user requirement specification (urs)
- software requirement specification (srs) - functional requirements external interface requirements system features and non-functional requirements. constraints, assumptions, and dependencies
- system requirement specification (sysrs) - system capabilities, interfaces, and user characteristics. it also may include policy requirements, regulation requirements, personnel requirements, performance requirements, security requirements, and system acceptance criteria, hardware

*** application architecture 
- component based and service oriented architecture - characteristics
- component
  - component - individual unit of encapsulated functionality
  - char'c: reusable, replaceable, independent, extensible, encapsulated, non-context specific
  - eg: api, data access object, controller
- services
  - designed to be deployed independently and reused by multiple systems - loosely coupled and supports distributed systems 
  - has one unique, always running instance with whom multiple clients communicate 
  - eg: check credit rating, calc emi, process mortgage application 
- application implementation layers: object / class layer > component layer > service layer  
- distributed system - characteristics
  - multiple services located on different machines
  - services coordinate interactions via a communication protocol such as http
  - appears to the end user as a single coherant system 
  - char'c: share resources, fault-tolerant, multiple activities run concurrently, scalable, runs on a variety of computers, polyglot programming 
  - nodes: any device on a network that can recognize, process, and transmit data to other nodes on the network.

*** production deployment components
- components: presentation tier -> firewall -> web server / load balancer -> application server / proxy -> data server / high availablity data replica 
- firewall is a security device that monitors traffic between networks
- load balancer distributes network traffic amongst servers
- web server delivers content such as web pages, files, images, and videos to a client
- application server runs business logic and provides the application to the client rather than the client running the app on their own machine
- database server stores and controls the flow of data through a database management system.
- proxy - load balancing, system optimization, caching, acting as a firewall, obscuring the source of the request, encryption, scanning for malware, and more. improve efficiency, privacy, and security of data flowing through a network
